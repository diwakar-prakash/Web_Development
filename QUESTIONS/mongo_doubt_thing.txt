Absolutely, brother Diwakar üëä ‚Äî let‚Äôs clear this up properly because this topic is often confused between **MongoDB** and **HTTP methods (GET, POST, PUT, DELETE)**.

MongoDB itself doesn‚Äôt have ‚ÄúGET, POST, PUT, DELETE‚Äù methods ‚Äî those are **HTTP verbs**, used in REST APIs (e.g., when you‚Äôre building Express routes in Node.js).

But in your **backend**, these HTTP methods map to **MongoDB operations**.
So let‚Äôs bridge both worlds ‚Äî I‚Äôll show you what **GET, POST, PUT, DELETE** do in REST, and which **MongoDB methods** they use internally üëá

---

### ‚öôÔ∏è 1. **GET ‚Üí Read data**

When you **fetch** data from MongoDB.

#### üìò MongoDB methods:

```js
// Get all documents
db.collection.find({})

// Get one document
db.collection.findOne({ _id: ObjectId("123...") })
```

#### Example in Express:

```js
app.get("/users", async (req, res) => {
  const users = await User.find(); // MongoDB find()
  res.json(users);
});
```

**Real-world:** When you open your profile page, your browser sends a GET request ‚Üí server runs `.find()` to get your data.

---

### üü¢ 2. **POST ‚Üí Create data**

When you **insert** new data into MongoDB.

#### üìò MongoDB methods:

```js
// Insert single document
db.collection.insertOne({ name: "Diwakar", age: 20 });

// Insert multiple documents
db.collection.insertMany([{ name: "Harsh" }, { name: "Aman" }]);
```

#### Example in Express:

```js
app.post("/users", async (req, res) => {
  const newUser = await User.create(req.body); // MongoDB insertOne()
  res.json(newUser);
});
```

**Real-world:** When you sign up on a site, your info is sent with POST ‚Üí backend does `.insertOne()`.

---

### üü† 3. **PUT / PATCH ‚Üí Update data**

When you **update** existing data in MongoDB.

#### üìò MongoDB methods:

```js
// Update one
db.collection.updateOne(
  { _id: ObjectId("123...") },
  { $set: { age: 21 } }
);

// Update many
db.collection.updateMany(
  { country: "India" },
  { $set: { verified: true } }
);

// Or with Mongoose
await User.findByIdAndUpdate(id, req.body, { new: true });
```

#### Example in Express:

```js
app.put("/users/:id", async (req, res) => {
  const updatedUser = await User.findByIdAndUpdate(req.params.id, req.body, { new: true });
  res.json(updatedUser);
});
```

**Real-world:** When you change your profile photo, a PUT request is made ‚Üí MongoDB updates that field.

---

### üî¥ 4. **DELETE ‚Üí Remove data**

When you **delete** documents from MongoDB.

#### üìò MongoDB methods:

```js
// Delete one
db.collection.deleteOne({ _id: ObjectId("123...") });

// Delete many
db.collection.deleteMany({ isInactive: true });
```

#### Example in Express:

```js
app.delete("/users/:id", async (req, res) => {
  await User.findByIdAndDelete(req.params.id); // MongoDB deleteOne()
  res.json({ message: "User deleted" });
});
```

**Real-world:** When you delete a post, the DELETE request triggers `.deleteOne()` on the backend.

---

### üîÅ Quick Summary Table

| HTTP Method   | CRUD Operation | MongoDB Method(s)                                    | Description          |
| ------------- | -------------- | ---------------------------------------------------- | -------------------- |
| **GET**       | Read           | `find()`, `findOne()`                                | Fetch data           |
| **POST**      | Create         | `insertOne()`, `insertMany()`, `create()`            | Add new data         |
| **PUT/PATCH** | Update         | `updateOne()`, `updateMany()`, `findByIdAndUpdate()` | Modify existing data |
| **DELETE**    | Delete         | `deleteOne()`, `deleteMany()`, `findByIdAndDelete()` | Remove data          |

## ‚öôÔ∏è Mongoose CRUD Operations with Examples

Let‚Äôs assume we have a simple **User model** üëá

```js
// models/User.js
const mongoose = require("mongoose");

const userSchema = new mongoose.Schema({
  name: String,
  age: Number,
  email: String
});

const User = mongoose.model("User", userSchema);

module.exports = User;
```

Now we‚Äôll use this `User` model to perform **GET, POST, PUT, and DELETE** operations.

---

### üü¢ 1. **POST ‚Üí Create Data**

Used to **add new documents** in the database.

#### üìò Mongoose methods:

* `User.create()`
* `new User().save()`

#### Example:

```js
app.post("/users", async (req, res) => {
  try {
    const newUser = await User.create({
      name: "Diwakar",
      age: 20,
      email: "diwakar@example.com"
    });
    res.status(201).json(newUser);
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});
```

**OR another way:**

```js
const user = new User({ name: "Harsh", age: 21 });
await user.save();
```

‚úÖ *Creates and saves a new user in MongoDB.*

---

### üîµ 2. **GET ‚Üí Read Data**

Used to **fetch** data from MongoDB.

#### üìò Mongoose methods:

* `User.find()` ‚Üí Get all documents
* `User.findById(id)` ‚Üí Get one by ID
* `User.findOne(condition)` ‚Üí Get one by condition
* `User.findByIdAndSelect(fields)` ‚Üí Select specific fields

#### Example:

```js
// Get all users
app.get("/users", async (req, res) => {
  try {
    const users = await User.find();
    res.json(users);
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});

// Get single user by ID
app.get("/users/:id", async (req, res) => {
  try {
    const user = await User.findById(req.params.id);
    if (!user) return res.status(404).json({ message: "User not found" });
    res.json(user);
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});
```

‚úÖ *Retrieves one or multiple documents.*

---

### üü† 3. **PUT / PATCH ‚Üí Update Data**

Used to **modify** existing documents.

#### üìò Mongoose methods:

* `User.updateOne(filter, update)`
* `User.updateMany(filter, update)`
* `User.findByIdAndUpdate(id, update, options)`
* `User.findOneAndUpdate(filter, update, options)`

#### Example:

```js
app.put("/users/:id", async (req, res) => {
  try {
    const updatedUser = await User.findByIdAndUpdate(
      req.params.id,
      req.body,
      { new: true } // returns updated document
    );
    if (!updatedUser) return res.status(404).json({ message: "User not found" });
    res.json(updatedUser);
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});
```

‚úÖ *Updates an existing user and returns the latest version.*

---

### üî¥ 4. **DELETE ‚Üí Remove Data**

Used to **delete** documents from the database.

#### üìò Mongoose methods:

* `User.deleteOne(filter)`
* `User.deleteMany(filter)`
* `User.findByIdAndDelete(id)`
* `User.findOneAndDelete(filter)`

#### Example:

```js
app.delete("/users/:id", async (req, res) => {
  try {
    const deletedUser = await User.findByIdAndDelete(req.params.id);
    if (!deletedUser) return res.status(404).json({ message: "User not found" });
    res.json({ message: "User deleted successfully" });
  } catch (error) {
    res.status(500).json({ message: error.message });
  }
});
```

‚úÖ *Deletes the user whose ID matches the given one.*

### üß† Bonus: Extra Mongoose Query Helpers

These are not tied to HTTP verbs directly, but they‚Äôre **super useful**:

| Method                | Description                        | Example                            |
| --------------------- | ---------------------------------- | ---------------------------------- |
| `.select("name age")` | Choose specific fields             | `User.find().select("name age")`   |
| `.sort({ age: -1 })`  | Sort results                       | `User.find().sort({ age: -1 })`    |
| `.limit(5)`           | Limit number of results            | `User.find().limit(5)`             |
| `.skip(10)`           | Skip first 10 results (pagination) | `User.find().skip(10)`             |
| `.countDocuments()`   | Count documents                    | `User.countDocuments({ age: 20 })` |
